Raising a Ticket
All support issues must be raised via Reach Central. Direct emails to engineers are not supported unless escalating a P1 or P2 incident.
How to raise a ticket in Reach Central
1

Log in to Reach Central

Open Reach Central using your client credentials. Contact your account manager if you need access.

2

Click Create Ticket

The create ticket form will open. Fill in all fields carefully - more detail means faster resolution.

3

Select issue type and urgency

Choose Incident for outages. Set Urgency to match the priority: P1 = Critical, P2 = High, P3 = Medium, P4 = Low.

4

Attach supporting files

Upload screenshots, error logs, or XMLs. Attachments significantly speed up resolution.

5

Submit and track

You will receive a ticket reference number. Monitor all updates and communications in Reach Central.

Escalation process
Step-by-step escalation
1

Detect and classify

Identify the priority using the definitions on the Overview tab. Full service down or widespread user impact = P1 immediately.

2

Complete the pre-escalation checklist

Run through all troubleshooting steps above. Document what was checked and the outcome of each step.

3

Raise a ticket in Reach Central

Create an Incident ticket with the correct priority. The SLA clock starts from ticket creation time.

4

Send escalation email for P1 / P2

Email rmts@reachmobile.com with subject: [P1] ClientName - Issue - Ticket #XXXX. Include affected service, user impact, and ticket reference.

5

Await acknowledgement

The team will respond within the SLA window. All further updates go through Reach Central - do not send duplicate emails.

6

Post-incident review (P1 only)

A root cause analysis (RCA) will be shared within 48 hours of P1 resolution, attached to the original ticket.
